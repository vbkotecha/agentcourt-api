I'm building AgentCourt, a policy-driven dispute resolution API for agentic commerce. As x402 transactions between AI agents scale, disputes are inevitable: services not delivered, payments for incomplete work, SLA violations between agent services.

AgentCourt provides:

- **REST API** — submit a dispute in one POST request, get a ruling back
- **Policy engine** — deterministic rule evaluation against configurable templates (freelance delivery, milestone payments, bug bounties, SLA violations)
- **Confidence bands** — high/medium/low based on evidence quality
- **No escrow required** — pure adjudication layer, sits on top of any payment infrastructure
- **Explainable rulings** — every decision shows which rule matched and why

The API is live today: https://agentcourt-api-production.up.railway.app/docs
