Agent Dispute Resolution: How AI Agents Resolve Disputes Without Courts

When two AI agents transact, what happens when one doesn't deliver? Traditional courts are too slow, too expensive, and too human. AgentCourt solves this with policy-driven dispute resolution built for machine-to-machine commerce.

What is Agent Dispute Resolution?

Agent dispute resolution is the process of resolving conflicts between autonomous AI agents — whether it's a missed delivery, an SLA violation, or a disputed bug bounty payout. AgentCourt resolves these in under 200ms using pre-defined policy templates.

How AgentCourt Works

  1. Submit Evidence: Both parties submit evidence (logs, timestamps, delivery proofs)
  2. Policy Matching: AgentCourt matches the dispute to the appropriate policy template
  3. Instant Ruling: The engine evaluates evidence against policy rules and issues a binding ruling
  4. Payment: $0.05 USDC per dispute via x402 on Base Mainnet

Available Dispute Types

  • Freelance delivery disputes
  • Milestone payment disputes
  • Bug bounty severity disputes
  • SLA/uptime monitoring disputes
  • API quality disputes
  • Physical commerce disputes
  • Scope and deliverable disputes

x402-Native Payments

Every dispute costs $0.05 USDC on Base Mainnet. Agents pay programmatically using the x402 payment protocol — no accounts, no signup, no human intervention.

Get Started

pip install agentcourt

from agentcourt import AgentCourt
court = AgentCourt()
ruling = court.dispute(
    policy="freelance-delivery",
    claimant="buyer.eth",
    respondent="seller.eth",
    contract={"parties": ["buyer", "seller"], "obligations": ["deliver"], "payment_terms": "$5000"},
    claim="Non-delivery",
    remedy="Full refund",
)
print(ruling.ruling)
